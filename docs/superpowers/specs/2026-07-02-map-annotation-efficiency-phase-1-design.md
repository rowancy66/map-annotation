# 地图标注效率增强第一期设计

## 概述

第一期只解决单地图编辑器中的高频效率问题，不扩展新的存储系统或复杂后端查询。范围限定为：可组合的本地筛选面板、批量移动分组/删除/字段值更新，以及导入时按编号默认更新旧标注。

## 目标

- 让管理员能在单张地图内快速定位目标标注
- 让重复性批量维护操作减少手工逐条编辑
- 让同一份业务表格可以反复导入并持续更新地图数据
- 保持改动最小，不引入附件、历史记录、服务端分页或复杂查询 DSL

## 功能范围

### 1. 查询面板

在地图编辑器左侧列表模式中，将当前简单搜索升级为可展开的查询面板，仍然使用前端本地筛选。

支持条件：

- 关键词：匹配名称、描述、自定义字段值
- 分组：单选当前分组，支持全部标注
- 标注类型：点 / 线 / 面 / 文字，多选
- 自定义字段筛选：
  - 文本/选择：包含或精确匹配
  - 数字：最小值 / 最大值
  - 日期：开始日期 / 结束日期

行为约束：

- 多条件之间使用 AND 关系
- 面板状态保存在当前页面状态中，不做持久化
- 地图与列表共享同一份筛选结果
- 面板可一键清空全部条件

### 2. 批量编辑与批量删除

保留现有批量选择模式，扩展三类操作：

- 批量移动到分组
- 批量删除
- 批量更新某一个自定义字段的值

行为约束：

- 仅允许对当前筛选结果中的选中项执行
- 批量改字段值一次只修改一个字段，避免首期界面过重
- 批量更新时，若标注缺少该字段，则补写对应 `custom_fields`
- 所有批量操作都需要确认
- 完成后给出操作结果数量

### 3. 导入增强

导入逻辑继续限定为点标注，但新增预览结果分类，并按编号默认更新旧标注。

编号规则：

- 优先使用映射到 `name` 的列作为编号
- 同一地图内，若已有点标注名称与导入编号相同，则执行更新
- 若不存在同编号标注，则新增

更新规则：

- 导入值覆盖旧值
- 空字符串或空值不覆盖已有字段值
- 经纬度存在且合法时覆盖旧坐标
- 描述存在时覆盖旧描述
- 自定义字段按映射结果逐项覆盖

预览结果需要展示：

- 预计新增数量
- 预计更新数量
- 缺少有效经纬度的行数
- 缺少编号时是否将生成默认名称

## 数据边界

- 不新增数据库表
- 不引入历史记录、撤销或附件系统
- 不新增服务端复杂筛选接口
- 服务端新增的仅是批量更新字段值、批量移动分组增强校验，以及导入按编号更新逻辑

必要时新增小型 UI 子组件以避免 `AdminEditor.tsx` 继续膨胀。

## 验证标准

- 能通过组合条件快速过滤标注，地图和列表同步变化
- 能对选中标注批量移动分组、批量删除、批量改某字段值
- 再次导入同编号数据时，旧点标注被更新而不是重复新增
- 导入前能看到新增/更新数量预估
- `npm run lint` 通过
