# 恺烨地图工作台对齐设计

## 概述

本次改造目标不是复制兰图绘的完整企业后台，而是借用其“地图即工作台”的布局逻辑，对齐单屏作业体验，同时替换为恺烨控股自身的品牌气质。

本次范围覆盖两个页面：

- `src/app/admin/AdminEditor.tsx`
- `src/app/map/[id]/page.tsx`

改造后的两个页面需要共享同一套结构语言：

- 顶部横向主工具带
- 左侧单工作面板
- 中央全屏地图画布
- 地图上沿悬浮工具
- 右上角精简管理入口

其中：

- `/admin` 定位为“编辑工作台”
- `/map/[id]` 定位为“阅览工作台”

## 目标

- 让后台编辑页更接近专业地图作业台，而不是内容型管理页
- 让前台只读页与后台共享同一产品语言，而不是两套割裂体验
- 保留现有轻量产品边界，不引入评论、协作、统计、帮助等重后台模块
- 用恺烨控股品牌色替代当前偏纸感的档案馆风格
- 在不重构核心数据模型的前提下，通过布局和组件重排提升专业感

## 非目标

- 不新增评论、统计、数据管理、协作、分享、帮助模块
- 不改造数据库结构
- 不引入新的权限模型
- 不重做标注绘制逻辑、导入导出逻辑或分组数据结构
- 不把左侧面板改成多标签工作区

## 设计原则

### 1. 地图优先

地图必须成为绝对视觉中心。页面应减少厚重卡片和过多包裹容器，优先保证地图面积、工具触达效率和状态感知。

### 2. 单屏作业

高频动作必须集中在首屏：

- 搜索
- 点线面绘制
- 标注定位
- 地图管理
- 导入导出
- 显示设置

避免让用户在不同页面和弹层之间频繁跳转。

### 3. 品牌替换而非照搬

结构参考兰图绘，但视觉气质必须转为恺烨控股：

- 深绿主色
- 香槟金强调
- 米白与浅砂背景
- 稳重、克制、专业

不能保留通用互联网后台的蓝白感，也不能继续维持当前较重的纸质档案风。

### 4. 最小必要改造

优先通过布局重排、样式令牌更新、现有组件重新编排达成目标。只在必要时拆分小型视图组件，不做超范围抽象。

## 品牌与视觉系统

### 品牌方向

参考恺烨控股 logo，整体视觉定位为“地产/资管风格的专业地图工作台”。

关键词：

- 稳定
- 秩序
- 克制
- 精准

### 配色建议

建议将全局变量调整为以下关系：

- 主品牌色：深绿，用于主按钮、选中态、工具激活态、顶部重点按钮
- 强调色：香槟金，用于描边高亮、选中标签、关键数字、焦点态
- 基底色：米白/浅砂，用于页面背景和侧栏底色
- 正文字：深墨灰，用于正文和主信息
- 弱文字：灰棕，用于说明信息和次级标签

视觉层级应从“高对比大卡片”调整为“低对比薄层次”：

- 页面对比更轻
- 面板边界更薄
- 顶栏更扁平
- 工具条更悬浮
- 选中态依靠描边、阴影和色块，而不是大面积厚背景

### 组件气质调整

- `paper-panel`：从厚玻璃卡片改为更轻的工作区底板
- `paper-card`：减少在工作台主区域的使用，仅保留在必要详情区
- `soft-pill`：用于状态计数与轻筛选标签
- `ghost-button`：改为更接近工具按钮的扁平控制
- 新增统一的悬浮工具条和工作面板样式类

## 信息架构

### 共用骨架

`/admin` 与 `/map/[id]` 共享以下布局骨架：

1. 页面整体全屏
2. 顶部一条横向主工具带
3. 左侧单工作面板
4. 中央地图画布
5. 地图画布上沿的悬浮工具区
6. 地图右上角的轻量操作入口

### 左侧面板策略

左侧保留单面板，不做兰图绘式多标签切换。

原因：

- 当前产品能力集中，单面板足以承载
- 多标签会引入额外状态复杂度
- 用户已习惯当前单侧栏模式
- 本轮目标是像“工作台”，不是扩展成复杂中台

因此只调整左侧面板内部结构，不改变其基本交互模型。

## `/admin` 页面设计

### 页面定位

`/admin?mapId=...` 是主编辑工作台，应对齐“地图作业台”的心智模型。

### 顶部主工具带

顶部工具带分为三段：

#### 左侧

- 返回地图列表
- 返回前台
- 项目标识
- 当前地图名称
- 管理态标识

该区域应更简洁，减少当前标题区域的内容密度。

#### 中部

中部改为全局搜索入口，优先级高于当前居中的绘制工具栏。

搜索范围保持现有能力边界：

- 标注名称
- 描述
- 自定义字段值

搜索框应具备：

- 明确占位文案
- 清空能力
- 结果状态反馈

#### 右侧

只保留高频、与当前产品直接相关的入口：

- 地图管理
- 导入
- 导出
- 显示设置
- 退出登录

不新增其他企业级功能入口。

### 地图上沿悬浮工具

点、线、面绘制工具从顶栏主中位移出，改为悬浮在地图区域上沿，形成更接近兰图绘的“贴地图操作”体验。

建议保留并重排：

- 点标注
- 线标注
- 面标注
- 显示名称开关
- 热力图开关（如当前能力仍保留）

工具条需要具备：

- 激活态明显
- 与地图边缘有安全留白
- 在移动端可换行或折叠

### 左侧工作面板

左侧面板内部改成三层：

#### 第一层：项目与搜索摘要

- 当前地图名称
- 标注总数
- 快捷搜索框

#### 第二层：轻切换控制

保留当前已有的核心切换，但只做轻量分段，不做复杂 tab：

- 标注列表
- 分组视图
- 筛选面板

表现形式可以是 segmented control，而不是厚标签页。

#### 第三层：主体内容区

根据第二层切换显示：

- 标注列表
- 分组树
- 筛选条件

当前批量选择、批量移动、批量删除、批量字段更新能力仍保留在该区域内，但视觉上应做成列表上下文动作，而不是独立重面板。

### 详情与设置区

现有详情卡片和字段模板管理器继续保留，但应避免同时与左侧面板争夺层级。

建议：

- 标注详情仍使用右侧或浮层方式呈现
- 地图设置保留抽屉或弹层，不作为常驻大区块

目标是避免页面中心被多个厚面板同时切碎。

## `/map/[id]` 页面设计

### 页面定位

`/map/[id]` 是轻量阅览工作台，应与 `/admin` 视觉同源，但去掉编辑负担。

### 顶部主工具带

顶部保留与后台一致的结构语言，但内容更克制。

建议包括：

- 返回入口
- 项目名称
- 搜索框
- 标注数量
- 地图管理入口
- 后台管理入口

“地图管理入口”在只读页中应表现为导航入口，而不是暴露后台控制项。

### 左侧工作面板

只读页左侧仍保留单面板，名称可继续使用“标注索引”。

内部结构建议为：

1. 顶部摘要
2. 搜索框
3. 类型计数
4. 标注列表

不引入多标签，不新增复杂筛选流程。

### 地图操作区

只读页地图区域仅保留阅览相关控制：

- 显示名称开关
- 图例
- 图层切换
- 缩放控件

不显示点线面绘制工具，不出现编辑态按钮。

### 详情展示

标注详情卡片继续存在，但应沿用与后台一致的样式系统：

- 更轻边框
- 更薄背景
- 统一标题层级

确保只读页不是另一套设计体系。

- 工作台顶栏

- 地图悬浮工具带

但不应为了抽象而抽象。

## 样式系统改造范围

建议优先调整：

- `src/app/globals.css`

需要新增或重定义的样式包括：

- 工作台背景层
- 顶部工具带
- 左侧面板
- 悬浮工具条
- 地图管理入口按钮
- 选中态、激活态、焦点态

现有全局变量应从“档案馆纸感”向“恺烨品牌工作台”迁移，但仍保持可读性和地图兼容性。

## 响应式策略

### 桌面端

桌面端优先完整呈现工作台形态：

- 左侧面板常驻
- 顶部工具带完整展示
- 地图工具悬浮显示

### 平板与小屏

小屏下允许退化：

- 左侧面板可折叠
- 顶部部分文字按钮缩成图标
- 地图上沿工具条允许换行或横向滚动

但不改变主结构，不另做第二套布局体系。

## 实施顺序

### 第一阶段：骨架与品牌

- 重做 `/admin` 与 `/map/[id]` 的布局骨架
- 更新全局色板和核心工作台样式
- 把绘制工具移到地图悬浮区
- 加入顶部搜索主入口和地图管理入口

### 第二阶段：组件贴合

- 重排左侧面板内部结构
- 调整标注列表卡片的密度、选中态和层级
- 统一详情卡片与设置面板样式
- 校准桌面与移动端适配

## 验证标准

- `/admin` 首屏明显呈现“地图作业台”而非内容管理页
- `/map/[id]` 与 `/admin` 在骨架和品牌气质上保持统一
- 顶部只出现与当前产品相关的核心入口，不出现评论、协作等重功能
- 左侧仍为单面板结构，没有引入多标签复杂度
- 深绿与香槟金成为主要品牌识别，而不是当前偏纸感配色
- 地图区域视觉权重高于侧栏和详情区
- 现有标注、分组、搜索、导入导出、设置能力保持可用

## 风险与约束

- `AdminEditor.tsx` 当前职责较多，布局重排时需要警惕文件继续膨胀
- `MapView.tsx` 已承载较多地图交互，新增悬浮工具布局时应优先通过外层容器解决，不要把展示逻辑继续压进地图核心逻辑
- 品牌色需要兼顾地图底图可读性，不能让深绿大面积压住地图主体
- 首期仅做结构和视觉对齐，不应顺手扩张为新功能开发
