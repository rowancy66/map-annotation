# 后台地图编辑页参考图对齐设计

## 目标

将后台地图编辑页整体重设计为“冷静专业版”地图工作台，对齐参考图的结构秩序、信息密度和工具分布方式，同时保留恺烨品牌的克制表达。目标不是直接复刻兰图绘，而是学习其骨架与交互组织方式，输出一套更符合当前产品身份的后台编辑界面。

本次重设计覆盖整个后台编辑页，包括顶部工具条、左侧一体化工作面板、地图内浮层控件、右侧轻量地图控件列，以及相关状态样式。前台页面、登录页、地图详情页不在本次范围内。

## 已确认决策

### 视觉方向

- 采用“冷静专业版”方向：白底、高密度、细边框、弱阴影、低圆角
- 以参考图的工具软件秩序为主，不使用大标题海报感和营销式装饰
- 恺烨绿只用于激活态、焦点态和少量强调，不做大面积铺色
- 地图应继续是页面主角，界面控件围绕地图工作流组织

### 对齐力度

- 采用“参考图骨架 + 恺烨品牌重译”
- 学习参考图的布局结构、工具条分层、左侧工作面板逻辑、右侧控件位置
- 不直接复刻参考图的品牌文字、装饰图标或无关业务模块

### 左侧栏决策

- 采用参考图式一体化工作面板
- 顶部标签按“地图 / 标注 / 分组 / 分类 / 筛选”组织
- 标签下承接分类快捷入口、筛选上下文和结果区
- 不再维持当前“分散表单块 + 多段面板”的结构

### 顶部工具条决策

- 顶部采用参考图式单行专业 GIS 工具条
- 搜索居中，工具横向连续排布
- 不使用当前多段分裂式按钮组布局

### 保留的顶栏功能

- 显示名称
- 智能标注
- 点标注
- 线标注
- 面标注
- 工具
- 出图 / 导出
- 刷新
- 地图管理入口
- 设置
- 退出 / 账户

### 不保留功能

- 不保留“定位城市”

### 地图右侧与浮层态度

- 右侧控件也向参考图靠齐
- 保留轻量竖向控件列，如图层、图例、缩放等
- 地图内浮层尽量压缩和规范，避免当前多块漂浮、层级冲突的问题

## 页面结构设计

### 1. 顶部主工具条

顶部改为单层横向工作条，高度控制在专业工具软件的常规密度内，整体从左到右分为五段：

1. 品牌与项目标识
2. 主搜索区
3. 地图显示与绘制工具
4. 系统操作工具
5. 账户区

#### 1.1 品牌与项目标识

左侧区域包含：

- 简化品牌识别
- 当前地图项目名称
- 次级说明文字（如“地图编辑”）

返回地图列表与返回前台不再占据工具条主视觉位置，可收敛为左端的轻量图标入口，避免干扰工具条主体节奏。

#### 1.2 主搜索区

搜索框位于顶部中央，宽度为整条工具带的视觉中心。它承担当前地图内标注搜索，不再和左侧筛选面板形成重复视觉中心。搜索框表现应接近参考图：长、薄、浅边框、低强调。

#### 1.3 地图显示与绘制工具

搜索框右侧开始连续排布工具：

- 显示名称
- 智能标注
- 点标注
- 线标注
- 面标注
- 工具

这些控件以“细分隔 + 同高按钮”组织。激活态仅用恺烨绿强调，不使用厚重底块。点/线/面作为主绘制操作，应具有更清晰的当前态反馈。

#### 1.4 系统操作工具

再往右依次为：

- 出图 / 导出
- 刷新
- 地图管理
- 设置

其中“地图管理”保留为功能入口，但视觉优先级低于绘制工具，不应做成独立大按钮块。

#### 1.5 账户区

最右侧为用户头像 / 账户 / 退出，表现参考图的轻量账户区逻辑，不再以大按钮形式打断工具条连续性。

## 2. 左侧一体化工作面板

左侧栏改造成参考图式工作面板，宽度更稳定，信息更集中。结构分为四层：

1. 顶部模式标签
2. 当前上下文工具区
3. 结果区 / 目录区
4. 空状态或操作提示

### 2.1 顶部模式标签

顶部使用一排轻量标签：

- 地图
- 标注
- 分组
- 分类
- 筛选

标签是模式切换入口，而不是当前那种“列表/分组”二元切换。标签应细、浅、靠上排列，接近参考图的密度。

### 2.2 当前上下文工具区

根据激活标签变化展示不同内容：

- `地图`：展示当前地图基础上下文
- `标注`：展示标注总览与目录
- `分组`：展示分组树及分组操作
- `分类`：展示分类快捷入口或预设集合
- `筛选`：展示关键词、类型、字段条件等筛选能力

本次重构不要求后端增加新的分类数据结构，可以先用前端组织方式承载当前已有分组、类型、筛选逻辑。

### 2.3 结果区 / 目录区

结果区承接当前标签的主要内容。例如：

- 标注模式下：标注列表
- 分组模式下：分组树
- 筛选模式下：筛选后的结果

当前卡片式列表项要改成更接近参考图的“目录行”风格：更扁、更轻、信息更压缩。

### 2.4 空状态

空状态采用参考图式简洁空面板，不要大图示或过强提示语。只保留必要说明，例如“暂无查询结果”。

## 3. 地图主体与地图内工具

### 3.1 地图主体

- 地图继续占据主体面积
- 优先使用白色矢量底图表现
- 页面其他区域应主动退后，让地图成为唯一大面积视觉焦点

### 3.2 地图内顶部辅助控件

地图内不再保留当前与顶部主工具条重复的主要绘制控件。地图顶部只保留辅助性质控件，例如：

- 地图内搜索
- 必要的局部辅助状态

如果某些绘制能力已经进入顶部主工具条，地图内同类入口应删减，避免重复。

### 3.3 地图状态提示

点/线/面绘制过程中的提示仍需保留，但样式要更轻量，避免当前过于悬浮和抢画面的提示块。提示可以靠近地图上方中部，但应更薄、更短。

## 4. 右侧地图控件列

右侧做成参考图式轻量竖向控件列，包括：

- 图层入口
- 图例入口
- 缩放控件
- 其他必要地图开关

这些控件应统一宽度、高度和留白，悬浮在地图右侧，弱阴影、细边框、低圆角。不要再出现多个风格不一致的右侧漂浮块。

## 5. 组件与现有逻辑映射

本次尽量复用现有业务逻辑，只重组结构和表现：

- `AdminEditor`：作为主页面骨架重构中心
- `AnnotationFilterPanel`：改造成适配左侧一体化面板的筛选子模块
- `GroupTree`：改造成目录式树结构视觉
- `DrawingToolbar`：能力拆解后，部分操作迁移到顶部主工具条；保留地图内辅助版本或最小版本
- `MapView`：保留现有地图交互逻辑，但调整地图内控件挂载位置与风格

## 6. 样式策略

全局样式需从“编辑台双层面板”进一步收束到“参考图式 GIS 工作台”：

- 更白的背景层
- 更轻的边框对比
- 更少阴影
- 更统一的按钮高度
- 更细的字重和更小的控件字号
- 统一控件直角或极低圆角

不允许出现：

- 大圆角
- 胖按钮
- 大标题口号
- 过重的绿色块
- 类模板站式的分段卡片感

## 7. 交互要求

- 顶部工具在激活时必须有清晰反馈
- 左侧标签切换必须稳定，不能引起布局跳动
- 绘制点/线/面时，当前模式应同时在顶部工具条和地图状态中可识别
- 右侧地图控件列不可与信息卡或其他悬浮面板冲突
- 地图管理入口仍然要易找，但不能破坏工具条节奏

## 8. 测试与验证

改造完成后，至少验证：

1. `/admin?mapId=...` 页面可正常加载
2. 顶部工具条在常见桌面宽度下不拥挤、不换行错乱
3. 左侧面板切换不同标签时内容正常
4. 点/线/面绘制模式可切换且仍然可用
5. 地图内搜索、右侧地图控件、信息卡之间没有层级冲突
6. 页面在构建和 lint 下通过

## 范围外

本次不处理：

- 前台页面风格同步
- 登录页
- 新后端字段或分类模型设计
- 评论、统计、数据管理、协作、分享、帮助等参考图中的额外模块
